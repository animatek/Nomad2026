Sistema de Temas — Nomad2026

Primera pregunta: ¿cuántos temas necesitas al lanzar esta feature?

A

2 temas: Classic + Dark

Solo los dos que describes. Classic = el look actual (verde menta, gris azulado). Dark = el tema carbón que defines arriba. Intercambiables en runtime.

Pros

  • Scope mínimo, entregable rápido
  • Toda la arquitectura queda lista para temas futuros

Contras

  • Sin temas adicionales por ahora
B

2 temas + estructura para temas personalizados

Classic + Dark como base, más un sistema para cargar temas desde archivo JSON/XML externo. Un usuario podría crear su propio tema.

Pros

  • Extensible por la comunidad
  • Posible exportar/importar temas

Contras

  • Más complejidad en el parser
  • Requiere validación de archivos externos
C

Solo Dark (eliminar el tema Classic)

El proyecto ya no se parece al editor Java. Hacer dark el único modo, más simple y coherente.

Pros

  • Sin complejidad de theming
  • Identidad visual única

Contras

  • Pierde referencia visual con el original
  • No reversible fácilmente
